# Todo List 應用程式

一個簡潔的待辦事項管理應用程式，使用純 JavaScript 開發，並整合 Google Sheets 作為雲端資料庫。

## ✨ 功能特色

- 📝 新增待辦事項、描述與優先權
- 📱 手機版採用清單主畫面與底部新增抽屜
- ✅ 勾選狀態、展開後可標記完成與刪除
- 📊 依優先權排序（無 > 高 > 中 > 低）
- 🎨 優先權色彩標示（紅/橘/綠/灰）
- 📅 自動記錄建立日期
- ☁️ Google Sheets 雲端同步
- ⚙️ 可自訂 Google Script URL
- 🔁 提供載入失敗重試與狀態訊息提示

## 🚀 快速開始

### 1. 建立 Google Sheets 試算表

1. 開啟 [Google Sheets](https://sheets.google.com) 並建立新的試算表
2. 在第一列（標題列）依序輸入以下欄位名稱：

| 欄位 | 名稱 | 說明 |
|------|------|------|
| A1 | `紀錄日期` | 待辦事項建立時間 |
| B1 | `待辦事項` | 待辦事項名稱 |
| C1 | `優先權` | 可填入：高、中、低 |
| D1 | `完成日期` | 完成時間（選填） |
| E1 | `檢查狀態` | 勾選框（會自動建立） |
| F1 | `備註` | 待辦事項描述 |
| G1 | `ID` | 系統自動產生的識別碼 |

> 💡 **提示**：只需要輸入標題列即可，資料會由應用程式自動填入。

### 2. 設定 Google Apps Script

1. 在試算表中，點選「**擴充功能**」>「**Apps Script**」
2. 刪除預設程式碼，複製 `google-apps-script.js` 的內容貼入
3. 點選「**儲存**」（Ctrl+S）
4. 點選「**部署**」>「**新增部署**」
5. 選擇「**網頁應用程式**」，設定：
   - 執行身分：**我**
   - 誰可以存取：**所有人**
6. 點選「**部署**」，完成授權流程
7. 複製顯示的 **Web App URL**

### 3. 設定前端應用

1. 開啟 `index.html`
2. 點擊右上角 ⚙️ 設定按鈕
3. 貼上 Web App URL 並儲存

### 4. 📱 手機版操作方式

1. 進入頁面後，主畫面會優先顯示待辦清單。
2. 點擊底部 `新增待辦` 按鈕，會打開底部抽屜表單。
3. 輸入待辦名稱、描述與優先權後送出，即可新增項目。
4. 點擊任一待辦列可展開詳細內容，並在展開區執行：
   - 修改優先權
   - 標記完成
   - 刪除待辦

### 5. 💻 本機桌面端使用方式

本應用程式設計為純靜態網頁，無需安裝 Node.js 或架設伺服器即可在桌面端使用：

1. **直接開啟**：找到資料夾中的 `index.html` 檔案，雙擊直接以瀏覽器（如 Chrome, Edge）開啟。
2. **建立捷徑**：右鍵點擊 `index.html` >「傳送到」>「桌面（建立捷徑）」，之後即可像應用程式一樣快速啟動。
3. **資料同步**：只要設定好 Google Script URL，無論在哪台電腦開啟此檔案，都能同步存取您的雲端待辦清單。

桌面版維持上方可直接輸入的表單；手機版則改為底部抽屜式新增流程。

### 6. 🧪 開發與測試

若要執行自動化測試，請先安裝 Node.js，然後在專案目錄執行：

```bash
npm install
npx playwright install chromium
```

常用測試指令：

```bash
# 執行全部 E2E 測試
npm run test:e2e

# 以 headed 模式執行
npm run test:e2e:headed

# 只跑手機版
npx playwright test tests/mobile-ui.spec.js --project=mobile-chromium

# 只跑桌面版
npx playwright test tests/mobile-ui.spec.js --project=desktop-chromium
```

若要以本機靜態伺服器手動檢查畫面，可執行：

```bash
npx http-server . -p 4173 -c-1
```

然後開啟 `http://127.0.0.1:4173`。

## 📊 Google Sheets 欄位對應

| 欄位 | 內容 |
|------|------|
| A - 紀錄日期 | 自動填入 `YYYY-MM-DD HH:mm:ss` |
| B - 待辦事項 | Todo 名稱 |
| C - 優先權 | 高/中/低 |
| D - 完成日期 | （暫未使用） |
| E - 檢查狀態 | 勾選狀態 |
| F - 備註 | Todo 描述 |
| G - ID | 唯一識別碼（自動產生） |

## ⌨️ 快捷鍵

| 快捷鍵 | 功能 |
|--------|------|
| `Enter` | 在名稱輸入框送出 |
| `Ctrl+Enter` | 在任意輸入框送出 |
| `Escape` | 手機版關閉新增抽屜 |

## 🧭 介面說明

### 手機版

- 主畫面以待辦清單為主。
- 底部固定 `新增待辦` 按鈕。
- 新增表單以底部抽屜方式顯示。
- `完成`、`刪除` 與優先權調整位於展開區，避免主列過度擁擠。
- 錯誤與成功訊息以底部訊息條顯示。

### 桌面版

- 保留上方直接輸入的表單。
- 空狀態會引導使用者使用上方表單新增第一筆待辦。
- 同樣支援展開待辦、調整優先權、標記完成與刪除。

## 🔒 安全性說明

目前 Web App 設定為「所有人都能存取」，URL 本身即為存取憑證。請勿將 URL 分享給不信任的人。
